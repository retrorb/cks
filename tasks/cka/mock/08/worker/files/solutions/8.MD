# Task 8 — Scheduling: debug heavy-app (4 pts)

## Problem
Deployment `heavy-app` pods are Pending due to excessive resource requests. Save events, scale down, reduce resources.

## Solution

```bash
# Save events
kubectl get events -n resource-debug-ns --sort-by=.lastTimestamp > /var/work/tests/artifacts/8/events.txt

# Scale to 1 replica
kubectl scale deployment heavy-app -n resource-debug-ns --replicas=1

# Reduce resources
kubectl patch deployment heavy-app -n resource-debug-ns --type='json' \
  -p='[
    {"op":"replace","path":"/spec/template/spec/containers/0/resources/requests/cpu","value":"200m"},
    {"op":"replace","path":"/spec/template/spec/containers/0/resources/requests/memory","value":"256Mi"},
    {"op":"replace","path":"/spec/template/spec/containers/0/resources/limits/cpu","value":"200m"},
    {"op":"replace","path":"/spec/template/spec/containers/0/resources/limits/memory","value":"256Mi"}
  ]'
```

Verify:
```bash
kubectl get pods -n resource-debug-ns
kubectl get deployment heavy-app -n resource-debug-ns -o yaml | grep -A5 resources
cat /var/work/tests/artifacts/8/events.txt
```
