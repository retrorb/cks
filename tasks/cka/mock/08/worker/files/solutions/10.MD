# Task 10 — CNI fix: expand Calico IPPool (5 pts)

## Problem
A node's podCIDR falls outside the Calico IPPool range. Expand the IPPool CIDR.

## Solution

```bash
# Save current IPPool
kubectl get ippools -o yaml > /var/work/tests/artifacts/10/before.txt

# Edit IPPool to expand CIDR
kubectl get ippools -o name | head -1 | xargs kubectl edit
# Change cidr from 10.244.0.0/23 to 10.244.0.0/16

# Or use patch:
POOL_NAME=$(kubectl get ippools -o jsonpath='{.items[0].metadata.name}')
kubectl patch ippool $POOL_NAME --type='merge' -p '{"spec":{"cidr":"10.244.0.0/16"}}'

# Save updated IPPool
kubectl get ippools -o yaml > /var/work/tests/artifacts/10/after.txt
```

Verify:
```bash
kubectl get ippools -o yaml
diff /var/work/tests/artifacts/10/before.txt /var/work/tests/artifacts/10/after.txt
```
