# Task 13 — Native sidecar with restartPolicy Always (5 pts)

## Problem
Add a native sidecar (init container with `restartPolicy: Always`) to the `web-deploy` Deployment.

## Solution

```bash
kubectl edit deployment web-deploy -n native-sidecar-ns

# Add the following under spec.template.spec:
# 1. Add initContainers section
# 2. Add volume app-logs
# 3. Add volumeMount to main container
```

Or apply the full patch:

```bash
kubectl patch deployment web-deploy -n native-sidecar-ns --type='json' -p='[
  {"op":"add","path":"/spec/template/spec/initContainers","value":[
    {
      "name":"log-shipper",
      "image":"busybox",
      "command":["sh","-c","tail -f /var/log/app/app.log"],
      "restartPolicy":"Always",
      "volumeMounts":[{"name":"app-logs","mountPath":"/var/log/app"}]
    }
  ]},
  {"op":"add","path":"/spec/template/spec/volumes","value":[
    {"name":"app-logs","emptyDir":{}}
  ]},
  {"op":"add","path":"/spec/template/spec/containers/0/volumeMounts","value":[
    {"name":"app-logs","mountPath":"/var/log/app"}
  ]}
]'
```

Verify:
```bash
kubectl get deployment web-deploy -n native-sidecar-ns -o yaml
kubectl get pods -n native-sidecar-ns
```
