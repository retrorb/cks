# Task 16 — LimitRange: create strict-limits (5 pts)

## Problem
Create a LimitRange, a passing pod, and capture the error from an over-limit pod.

## Solution

```bash
# Create LimitRange
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: LimitRange
metadata:
  name: strict-limits
  namespace: limit-ns
spec:
  limits:
  - type: Container
    max:
      cpu: "500m"
      memory: "512Mi"
    min:
      cpu: "50m"
      memory: "64Mi"
EOF

# Create passing pod
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: limit-test-pod
  namespace: limit-ns
spec:
  containers:
  - name: nginx
    image: nginx
    resources:
      requests:
        cpu: "100m"
        memory: "128Mi"
      limits:
        cpu: "100m"
        memory: "128Mi"
EOF

# Attempt over-limit pod and save error
kubectl run over-limit-pod --image=nginx -n limit-ns \
  --overrides='{"spec":{"containers":[{"name":"nginx","image":"nginx","resources":{"requests":{"cpu":"1","memory":"1Gi"},"limits":{"cpu":"1","memory":"1Gi"}}}]}}' \
  2>&1 > /var/work/tests/artifacts/16/error.txt || true

# If the above didn't capture the error, try:
kubectl run over-limit-pod2 --image=nginx -n limit-ns --dry-run=server \
  --overrides='{"spec":{"containers":[{"name":"nginx","image":"nginx","resources":{"requests":{"cpu":"1","memory":"1Gi"},"limits":{"cpu":"1","memory":"1Gi"}}}]}}' \
  2> /var/work/tests/artifacts/16/error.txt || true
```

Verify:
```bash
kubectl get limitrange -n limit-ns
kubectl get pod limit-test-pod -n limit-ns
cat /var/work/tests/artifacts/16/error.txt
```
