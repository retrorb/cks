# Task 3 — Fix extra-etcd static pod (7 pts)

## Problem
Static pod `extra-etcd` is CrashLoopBackOff due to port mismatch between `--listen-peer-urls` (2381) and `--initial-advertise-peer-urls` (2382), and wrong `--data-dir`.

## Solution

```bash
# SSH to master
ssh <master-ip>

# Edit the static pod manifest
sudo vi /etc/kubernetes/manifests/extra-etcd.yaml

# Fix 1: Change --initial-advertise-peer-urls port from 2382 to 2381
# Fix 2: Change --initial-cluster port from 2382 to 2381
# Fix 3: Change --data-dir from /var/lib/etcd-extra-wrong to /var/lib/etcd-extra

# The corrected lines should be:
# - --data-dir=/var/lib/etcd-extra
# - --listen-peer-urls=https://127.0.0.1:2381
# - --initial-advertise-peer-urls=https://127.0.0.1:2381
# - --initial-cluster=extra-etcd=https://127.0.0.1:2381

# Or use sed:
sudo sed -i 's|etcd-extra-wrong|etcd-extra|g' /etc/kubernetes/manifests/extra-etcd.yaml
sudo sed -i 's|2382|2381|g' /etc/kubernetes/manifests/extra-etcd.yaml
```

Verify:
```bash
kubectl get pods -n kube-system | grep extra-etcd
cat /etc/kubernetes/manifests/extra-etcd.yaml
```
