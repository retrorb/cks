# Task 12 — NetworkPolicy cross-NS AND+OR (7 pts)

## Problem
Allow ingress to `backend` pod from two sources using AND logic within each entry and OR between entries.

## Solution

```bash
cat <<EOF | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: backend-access
  namespace: cross-ns-pol
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          purpose: frontend
      podSelector:
        matchLabels:
          tier: presentation
    - namespaceSelector:
        matchLabels:
          purpose: admin
      podSelector:
        matchLabels:
          app: admin
EOF
```

Note: Each `from` entry has BOTH `namespaceSelector` AND `podSelector` at the same level (AND logic). The two entries in the `from` array are OR'd together.

Verify:
```bash
kubectl get networkpolicy -n cross-ns-pol -o yaml
```
