# Task 2 — CRD/CR: explain alert, create critical-alert (4 pts)

## Problem
The CRD `alerts.monitoring.example.com` exists. Explain the spec and create a Custom Resource.

## Solution

```bash
# Explain and save
kubectl explain alert.spec > /var/work/tests/artifacts/2/spec.txt

# Create CR
cat <<EOF | kubectl apply -f -
apiVersion: monitoring.example.com/v1
kind: Alert
metadata:
  name: critical-alert
  namespace: alert-ns
spec:
  severity: critical
  message: "CPU threshold exceeded"
  recipient: ops-team
EOF
```

Verify:
```bash
cat /var/work/tests/artifacts/2/spec.txt
kubectl get alert critical-alert -n alert-ns -o yaml
```
