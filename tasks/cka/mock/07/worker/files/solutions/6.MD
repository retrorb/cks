# Task 6 — NetworkPolicy deny-all + allow from role=api (5 pts)

## Problem
Create a deny-all ingress policy and then allow ingress only from pods with `role=api` on port 80.

## Solution

```bash
# Deny-all ingress
cat <<EOF | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all-ingress
  namespace: netpol-deny-ns
spec:
  podSelector: {}
  policyTypes:
  - Ingress
EOF

# Allow from role=api on port 80
cat <<EOF | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-api-ingress
  namespace: netpol-deny-ns
spec:
  podSelector:
    matchLabels:
      app: server
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: api
    ports:
    - protocol: TCP
      port: 80
EOF
```

Verify:
```bash
kubectl get networkpolicies -n netpol-deny-ns
kubectl describe networkpolicy allow-api-ingress -n netpol-deny-ns
```
