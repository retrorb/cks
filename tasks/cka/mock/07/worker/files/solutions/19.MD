# Task 19 — Sidecar + NetworkPolicy: secure-logger (6 pts)

## Problem
Create a pod with sidecar and a NetworkPolicy restricting ingress to `role=frontend` with DNS egress.

## Solution

```bash
# Create pod with sidecar
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: secure-logger
  namespace: compound-ns
  labels:
    app: secure-logger
spec:
  containers:
  - name: main
    image: nginx:alpine
    ports:
    - containerPort: 80
    volumeMounts:
    - name: shared-logs
      mountPath: /var/log/app
  - name: sidecar
    image: busybox
    command: ["sh", "-c", "tail -f /var/log/app/access.log"]
    volumeMounts:
    - name: shared-logs
      mountPath: /var/log/app
  volumes:
  - name: shared-logs
    emptyDir: {}
EOF

# Create NetworkPolicy
cat <<EOF | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: secure-logger-policy
  namespace: compound-ns
spec:
  podSelector:
    matchLabels:
      app: secure-logger
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: frontend
    ports:
    - protocol: TCP
      port: 80
  egress:
  - ports:
    - protocol: UDP
      port: 53
    - protocol: TCP
      port: 53
EOF
```

Verify:
```bash
kubectl get pod secure-logger -n compound-ns
kubectl get networkpolicy secure-logger-policy -n compound-ns -o yaml
```
