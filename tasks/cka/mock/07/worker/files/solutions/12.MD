# Task 12 — NetworkPolicy AND logic (6 pts)

## Problem
Create a NetworkPolicy that allows ingress to `db-pod` only from pods with `role=api` AND namespaces with `env=production` (AND logic — single from entry with both selectors).

## Solution

```bash
cat <<EOF | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: db-allow-api-production
  namespace: netpol-and-ns
spec:
  podSelector:
    matchLabels:
      role: db
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: api
      namespaceSelector:
        matchLabels:
          env: production
EOF
```

Note: The key is having both `podSelector` and `namespaceSelector` in the SAME `from` entry (no `-` before `namespaceSelector`). This creates AND logic. Separate entries with `-` would create OR logic.

Verify:
```bash
kubectl get networkpolicy db-allow-api-production -n netpol-and-ns -o yaml
```
