# Task 16 — ResourceQuota + LimitRange in team-ns (5 pts)

## Problem
Create ResourceQuota, LimitRange, and a pod that inherits default resource limits.

## Solution

```bash
# Create ResourceQuota
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: ResourceQuota
metadata:
  name: team-quota
  namespace: team-ns
spec:
  hard:
    pods: "10"
    cpu: "2"
    memory: "2Gi"
EOF

# Create LimitRange
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: LimitRange
metadata:
  name: team-limits
  namespace: team-ns
spec:
  limits:
  - type: Container
    default:
      cpu: "100m"
      memory: "128Mi"
    defaultRequest:
      cpu: "100m"
      memory: "128Mi"
EOF

# Create pod without explicit resources (will get defaults from LimitRange)
kubectl run quota-test-pod --image=nginx -n team-ns
```

Verify:
```bash
kubectl get resourcequota team-quota -n team-ns
kubectl get limitrange team-limits -n team-ns
kubectl get pod quota-test-pod -n team-ns -o jsonpath='{.spec.containers[0].resources}'
```
