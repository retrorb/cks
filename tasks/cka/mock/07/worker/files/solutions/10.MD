# Task 10 — Fix extra-apiserver static pod (6 pts)

## Problem
Static pod `extra-apiserver` is CrashLoopBackOff due to wrong etcd port 2399. Fix to 2379.

## Solution

```bash
# SSH to master node
MASTER=$(kubectl get nodes -l node-role.kubernetes.io/control-plane \
  -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')

ssh -o StrictHostKeyChecking=no -i /home/ubuntu/.ssh/id_rsa ubuntu@${MASTER} << 'REMOTE'
# Fix etcd port
sudo sed -i 's/2399/2379/g' /etc/kubernetes/manifests/extra-apiserver.yaml

# Wait for kubelet to pick up the change
sleep 10

# Save crictl ps output
sudo crictl ps > /var/work/tests/artifacts/10/crictl.txt
REMOTE
```

Verify:
```bash
ssh -o StrictHostKeyChecking=no -i /home/ubuntu/.ssh/id_rsa ubuntu@${MASTER} \
  "sudo grep etcd-servers /etc/kubernetes/manifests/extra-apiserver.yaml"
ssh -o StrictHostKeyChecking=no -i /home/ubuntu/.ssh/id_rsa ubuntu@${MASTER} \
  "cat /var/work/tests/artifacts/10/crictl.txt"
```
