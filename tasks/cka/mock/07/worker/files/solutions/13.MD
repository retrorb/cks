# Task 13 — Init container for DNS wait (4 pts)

## Problem
Add an init container to Deployment `main-deploy` that waits for `db-svc` DNS resolution.

## Solution

```bash
kubectl edit deploy main-deploy -n init-ns
```

Add the following under `spec.template.spec`:
```yaml
      initContainers:
      - name: init-wait
        image: busybox
        command: ['sh', '-c', 'until nslookup db-svc.init-ns.svc.cluster.local; do echo waiting for db-svc; sleep 2; done']
```

Or apply as a patch:
```bash
kubectl patch deployment main-deploy -n init-ns --type='json' -p='[
  {
    "op": "add",
    "path": "/spec/template/spec/initContainers",
    "value": [
      {
        "name": "init-wait",
        "image": "busybox",
        "command": ["sh", "-c", "until nslookup db-svc.init-ns.svc.cluster.local; do echo waiting for db-svc; sleep 2; done"]
      }
    ]
  }
]'
```

Verify:
```bash
kubectl get deploy main-deploy -n init-ns -o jsonpath='{.spec.template.spec.initContainers[*].name}'
kubectl get pods -n init-ns -l app=main-deploy
```
