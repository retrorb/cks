# Task 18 — Fix kubelet cert path on worker (6 pts)

## Problem
Worker node kubelet has wrong `tlsCertFile` path in config. Fix and restart.

## Solution

```bash
# Get worker node IP
WORKER=$(kubectl get nodes -l work_type=infra_core \
  -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')

ssh -o StrictHostKeyChecking=no -i /home/ubuntu/.ssh/id_rsa ubuntu@${WORKER} << 'REMOTE'
# Check current (broken) config
sudo grep tlsCertFile /var/lib/kubelet/config.yaml

# Find the correct cert file
ls -la /var/lib/kubelet/pki/

# Fix the cert path (use the actual kubelet client cert)
sudo sed -i 's|tlsCertFile:.*|tlsCertFile: /var/lib/kubelet/pki/kubelet-client-current.pem|' /var/lib/kubelet/config.yaml

# Restart kubelet
sudo systemctl restart kubelet
REMOTE
```

Verify:
```bash
kubectl get nodes
ssh -o StrictHostKeyChecking=no -i /home/ubuntu/.ssh/id_rsa ubuntu@${WORKER} \
  "sudo systemctl is-active kubelet"
```
