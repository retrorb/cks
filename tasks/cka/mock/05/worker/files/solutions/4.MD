## Task 4: Fix Wrong Image in Deployment

```bash
kubectl config use-context cluster1-admin@cluster1

# Fix the deployment image
kubectl set image deployment/bad-image-app bad-image-app=nginx:alpine -n image-ns

# Or edit directly
kubectl edit deployment bad-image-app -n image-ns
# Change image: nginx:doesnotexist999 to image: nginx:alpine

# Verify
kubectl get deployment bad-image-app -n image-ns
kubectl get pods -n image-ns
```

**Key concepts:**
- `ImagePullBackOff` or `ErrImagePull` indicates a bad image
- `kubectl set image` is the quickest way to update a deployment image
- The deployment will rolling-update pods automatically
