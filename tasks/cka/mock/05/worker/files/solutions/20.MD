## Task 20: Fix Service Selector and NetworkPolicy

```bash
kubectl config use-context cluster1-admin@cluster1

# Fix 1: Service selector
kubectl patch svc backend-svc -n final-ns --type='json' \
  -p='[{"op":"replace","path":"/spec/selector/app","value":"backend"}]'

# Fix 2: NetworkPolicy — allow ingress from client-pod
cat << 'EOF' | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: block-all
  namespace: final-ns
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: client
EOF

# Verify
kubectl get endpoints backend-svc -n final-ns
kubectl exec client-pod -n final-ns -- curl http://backend-svc -s --connect-timeout 3
```

**Key concepts:**
- Always check both the service selector AND NetworkPolicy when troubleshooting connectivity
- `ingress: []` means all ingress is blocked — add a rule with `from` to allow traffic
- Test connectivity with `kubectl exec <pod> -- curl <service-url>`
