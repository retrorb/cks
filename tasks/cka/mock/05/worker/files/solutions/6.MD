## Task 6: Fix PVC by Creating Missing StorageClass

```bash
kubectl config use-context cluster1-admin@cluster1

# Create the missing StorageClass
cat << 'EOF' | kubectl apply -f -
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: missing-sc
provisioner: kubernetes.io/no-provisioner
volumeBindingMode: WaitForFirstConsumer
EOF

# Verify PVC status
kubectl get pvc stuck-pvc -n pvc-ns
kubectl get sc missing-sc
```

**Key concepts:**
- PVCs stay in Pending if the referenced StorageClass doesn't exist
- `no-provisioner` means the PV must be created manually
- With `WaitForFirstConsumer`, binding is delayed until a pod uses the PVC
