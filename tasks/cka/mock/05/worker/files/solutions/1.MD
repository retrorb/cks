## Task 1: Remove NoSchedule Taint

```bash
kubectl config use-context cluster1-admin@cluster1

# Find the node with the taint
kubectl describe nodes -l work_type=infra_core | grep -A5 Taints

# Get node name
NODE=$(kubectl get nodes -l work_type=infra_core -o jsonpath='{.items[0].name}')

# Remove the taint (add - at the end)
kubectl taint node $NODE broken=true:NoSchedule-

# Verify
kubectl describe node $NODE | grep Taints
```

**Key concepts:**
- Taints prevent pods from being scheduled on nodes unless they have matching tolerations
- Remove a taint by appending `-` to the taint specification
- `NoSchedule` prevents new pods; `NoExecute` also evicts existing pods
