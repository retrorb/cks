## Task 13: Fix Service Selector

```bash
kubectl config use-context cluster1-admin@cluster1

# Edit the service to fix selector
kubectl edit svc no-ep-svc -n ep-ns
# Change: app: wrong-app -> app: real-app

# Or use patch
kubectl patch svc no-ep-svc -n ep-ns --type='json' \
  -p='[{"op":"replace","path":"/spec/selector/app","value":"real-app"}]'

# Verify endpoints
kubectl get endpoints no-ep-svc -n ep-ns
kubectl get pods -n ep-ns -l app=real-app
```

**Key concepts:**
- Service selector must match pod labels exactly
- No endpoints = selector mismatch or pods not running
- Check pod labels with `kubectl get pods --show-labels`
