## Task 3: Fix CrashLoopBackOff Pod

```bash
kubectl config use-context cluster1-admin@cluster1

# Check the current state
kubectl describe pod crash-app -n crash-ns

# Delete the existing broken pod and recreate with correct command
kubectl delete pod crash-app -n crash-ns

cat << 'EOF' | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: crash-app
  namespace: crash-ns
spec:
  containers:
  - name: crash-app
    image: nginx:alpine
    command: ["/bin/sh", "-c", "sleep 3600"]
EOF

# Verify
kubectl get pod crash-app -n crash-ns
```

**Key concepts:**
- `CrashLoopBackOff` means the container keeps failing to start
- Check logs with `kubectl logs crash-app -n crash-ns` and `kubectl describe pod`
- Pods cannot be edited in place if the command changes; delete and recreate
