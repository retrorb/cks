## Task 15: Fix NetworkPolicy to Allow DNS

```bash
kubectl config use-context cluster1-admin@cluster1

# Edit the NetworkPolicy to allow DNS egress
kubectl edit networkpolicy block-egress -n dns-block

# Add egress rules to allow DNS:
cat << 'EOF' | kubectl apply -f -
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: block-egress
  namespace: dns-block
spec:
  podSelector: {}
  policyTypes:
  - Egress
  egress:
  - ports:
    - port: 53
      protocol: UDP
    - port: 53
      protocol: TCP
EOF

# Verify DNS resolution works
kubectl exec no-dns-pod -n dns-block -- nslookup kubernetes.default.svc.cluster.local
```

**Key concepts:**
- DNS uses port 53 (both UDP and TCP)
- `egress: []` means no egress is allowed — must add rules
- Always allow DNS egress for pods that need to reach services by name
