## Task 11: Save Previous Container Logs

```bash
kubectl config use-context cluster1-admin@cluster1

# Save previous (crashed) container logs
mkdir -p /var/work/tests/artifacts/11
kubectl logs restart-pod -n restart-ns --previous --context cluster1-admin@cluster1 \
  > /var/work/tests/artifacts/11/prev-logs.txt

# Verify
cat /var/work/tests/artifacts/11/prev-logs.txt

# Check restart count
kubectl get pod restart-pod -n restart-ns
```

**Key concepts:**
- `--previous` retrieves logs from the previous (crashed) container instance
- Useful when a container crashes immediately after start
- Check restart count with `kubectl get pod` in the RESTARTS column
