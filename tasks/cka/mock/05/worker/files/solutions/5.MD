## Task 5: Fix OOMKilled Deployment

```bash
kubectl config use-context cluster1-admin@cluster1

# Edit the deployment to increase memory limit
kubectl edit deployment oom-app -n resource-ns
# Change: memory: "1Mi" -> memory: "64Mi" (both limits and requests)

# Or use patch
kubectl patch deployment oom-app -n resource-ns --type='json' \
  -p='[{"op":"replace","path":"/spec/template/spec/containers/0/resources/limits/memory","value":"64Mi"},{"op":"replace","path":"/spec/template/spec/containers/0/resources/requests/memory","value":"64Mi"}]'

# Verify
kubectl get deployment oom-app -n resource-ns
kubectl get pods -n resource-ns
```

**Key concepts:**
- `OOMKilled` means the container exceeded its memory limit
- Memory limits must be set high enough for the application to run
- Check with `kubectl describe pod <name>` to see OOMKilled reason
