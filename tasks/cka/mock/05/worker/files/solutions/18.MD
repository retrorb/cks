## Task 18: Fix Secret Key Name

```bash
kubectl config use-context cluster1-admin@cluster1

# View current secret keys
kubectl get secret app-secret -n secret-ns -o yaml

# Add the db-password key to the Secret
kubectl patch secret app-secret -n secret-ns --type='json' \
  -p='[{"op":"add","path":"/data/db-password","value":"cGFzc3dvcmQ="}]'
# "cGFzc3dvcmQ=" is base64 for "password"

# Or edit directly
kubectl edit secret app-secret -n secret-ns
# Add under data: db-password: cGFzc3dvcmQ=

# Recreate the pod
kubectl delete pod secret-pod -n secret-ns

cat << 'EOF' | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: secret-pod
  namespace: secret-ns
spec:
  containers:
  - name: secret-pod
    image: nginx:alpine
    env:
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: app-secret
          key: db-password
EOF

# Verify
kubectl get pod secret-pod -n secret-ns
```

**Key concepts:**
- Secret data must be base64 encoded
- `CreateContainerConfigError` often means a missing secret key
- Add missing keys rather than replacing the whole secret
