## Task 17: Fix ConfigMap Value

```bash
kubectl config use-context cluster1-admin@cluster1

# Edit the ConfigMap to fix DB_HOST value
kubectl edit configmap app-config -n config-ns
# Change: DB_HOST: "wrong-host" -> DB_HOST: "correct-db-host"

# Or use patch
kubectl patch configmap app-config -n config-ns --type='json' \
  -p='[{"op":"replace","path":"/data/DB_HOST","value":"correct-db-host"}]'

# Recreate the pod to pick up the new ConfigMap value
kubectl delete pod env-pod -n config-ns

cat << 'EOF' | kubectl apply -f -
apiVersion: v1
kind: Pod
metadata:
  name: env-pod
  namespace: config-ns
spec:
  containers:
  - name: env-pod
    image: nginx:alpine
    env:
    - name: DB_HOST
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: DB_HOST
EOF

# Verify
kubectl exec env-pod -n config-ns -- env | grep DB_HOST
```

**Key concepts:**
- Pods don't automatically pick up ConfigMap changes when using `valueFrom`
- Must recreate the pod for environment variable changes to take effect
- `envFrom` with `configMapRef` also requires pod restart for env vars
