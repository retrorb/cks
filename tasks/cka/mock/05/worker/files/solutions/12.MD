## Task 12: Fix Service targetPort

```bash
kubectl config use-context cluster1-admin@cluster1

# Edit the service to fix targetPort
kubectl edit svc wrong-port-svc -n svc-ns
# Change: targetPort: 9999 -> targetPort: 8080

# Or use patch
kubectl patch svc wrong-port-svc -n svc-ns --type='json' \
  -p='[{"op":"replace","path":"/spec/ports/0/targetPort","value":8080}]'

# Verify endpoints
kubectl get endpoints wrong-port-svc -n svc-ns
kubectl get svc wrong-port-svc -n svc-ns -o yaml
```

**Key concepts:**
- `targetPort` is the port on the pod/container, not the service port
- Wrong `targetPort` causes the service to have no healthy endpoints
- Endpoints show IPs only when the selector matches running pods with the correct port
