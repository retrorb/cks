## Task 2: Fix Broken Static Pod

```bash
kubectl config use-context cluster1-admin@cluster1

# SSH to master node
MASTER=$(kubectl get nodes -o jsonpath='{.items[?(@.metadata.labels.node-role\.kubernetes\.io/control-plane)].status.addresses[0].address}')

ssh -o StrictHostKeyChecking=no ubuntu@$MASTER

# Edit the static pod manifest (on master node)
sudo vi /etc/kubernetes/manifests/broken-scheduler.yaml

# Change:
#   - --config=/nonexistent/scheduler-config.yaml
# To:
#   - --config=/etc/kubernetes/scheduler-config.yaml

# The kubelet will automatically pick up the change and restart the pod
```

**Key concepts:**
- Static pods are managed by kubelet directly, not by the API server
- Changes to manifests in `/etc/kubernetes/manifests/` are picked up automatically
- The pod name is `<name>-<nodename>` for static pods
