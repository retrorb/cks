## Task 18: ExternalName Service

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: v1
kind: Service
metadata:
  name: external-db
  namespace: default
spec:
  type: ExternalName
  externalName: database.example.com
```

```bash
kubectl apply -f external-db.yaml

# Verify
kubectl get svc external-db
kubectl get svc external-db -o jsonpath='{.spec.externalName}'
```

**Key concepts:**
- ExternalName services create a CNAME record in cluster DNS
- No selector needed — traffic is forwarded to the external hostname
- Pods can use `external-db.default.svc.cluster.local` to reach `database.example.com`
- Useful for integrating external databases or services into the cluster service discovery
- No ClusterIP is allocated; `clusterIP` is empty
