## Task 8: Allow Specific Pod NetworkPolicy

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-client-to-server
  namespace: app-net
spec:
  podSelector:
    matchLabels:
      app: server
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: client
```

```bash
kubectl apply -f allow-client-to-server.yaml
```

**Verify:**
```bash
# allowed-client (role=client) should succeed:
kubectl exec allowed-client -n allowed-net -- curl http://server.app-net.svc -s --connect-timeout 3

# denied-client (role=other) should be blocked:
kubectl exec denied-client -n allowed-net -- curl http://server.app-net.svc -s --connect-timeout 2
```

**Key concepts:**
- `podSelector` on the policy targets which pods are protected
- `ingress.from.podSelector` specifies which pods are allowed as sources
- Only pods with `role=client` in any namespace can reach pods with `app=server`
