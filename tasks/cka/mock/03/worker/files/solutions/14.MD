## Task 14: Gateway API — Gateway + HTTPRoute

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: main-gateway
  namespace: gateway-ns
spec:
  gatewayClassName: nginx
  listeners:
  - name: http
    protocol: HTTP
    port: 80
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: app-route
  namespace: gateway-ns
spec:
  parentRefs:
  - name: main-gateway
  rules:
  - backendRefs:
    - name: gw-app-svc
      port: 80
```

```bash
kubectl apply -f gateway.yaml

# Verify
kubectl get gateway main-gateway -n gateway-ns
kubectl get httproute app-route -n gateway-ns
```

**Key concepts:**
- `Gateway` defines a listener (port, protocol) on a specific `GatewayClass`
- `HTTPRoute` defines routing rules and attaches to a `Gateway` via `parentRefs`
- Gateway API is the successor to Ingress — more expressive and role-oriented
- The GatewayClass `nginx` was pre-created; the candidate creates Gateway + HTTPRoute
