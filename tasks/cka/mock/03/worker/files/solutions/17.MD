## Task 17: API Access from Pod

```bash
kubectl config use-context cluster1-admin@cluster1

# Create ServiceAccount
kubectl create serviceaccount api-reader-sa -n api-access

# Bind to pre-existing ClusterRole pod-reader
kubectl create clusterrolebinding api-reader-binding \
  --clusterrole=pod-reader \
  --serviceaccount=api-access:api-reader-sa
```

```yaml
# Create the pod using the ServiceAccount
apiVersion: v1
kind: Pod
metadata:
  name: api-pod
  namespace: api-access
spec:
  serviceAccountName: api-reader-sa
  containers:
  - name: api-pod
    image: curlimages/curl:latest
    command: ["sleep", "3600"]
```

```bash
kubectl apply -f api-pod.yaml

# Wait for pod to be running
kubectl wait pod api-pod -n api-access --for=condition=Ready --timeout=60s

# Access the Kubernetes API from inside the pod and save the response
kubectl exec api-pod -n api-access -- sh -c \
  'curl -s https://kubernetes.default/api/v1/pods \
   -H "Authorization: Bearer $(cat /run/secrets/kubernetes.io/serviceaccount/token)" \
   -k' \
  > /var/work/tests/artifacts/17/response.txt

# Verify
grep -i "PodList" /var/work/tests/artifacts/17/response.txt
```

**Key concepts:**
- Pods use their ServiceAccount token (auto-mounted) to authenticate to the Kubernetes API
- The token is at `/run/secrets/kubernetes.io/serviceaccount/token`
- The CA cert is at `/run/secrets/kubernetes.io/serviceaccount/ca.crt`
- Use `-k` (insecure) or the CA cert for TLS validation
