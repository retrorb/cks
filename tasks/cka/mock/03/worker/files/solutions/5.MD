## Task 5: Fix Broken Service

```bash
kubectl config use-context cluster1-admin@cluster1

# Inspect the current (wrong) selector
kubectl get svc broken-service -n fix-me -o yaml

# The selector is "app: wrong-label" — fix it to match the deployment pods
kubectl patch svc broken-service -n fix-me \
  --type='json' \
  -p='[{"op":"replace","path":"/spec/selector/app","value":"broken-app"}]'

# Verify endpoints are now populated
kubectl get endpoints broken-service -n fix-me
```

Or edit directly:
```bash
kubectl edit svc broken-service -n fix-me
# Change selector.app from "wrong-label" to "broken-app"
```

**Verify the fix:**
```bash
kubectl get endpoints broken-service -n fix-me
# Should show pod IPs, not <none>
```

**Key concepts:**
- A service with no matching pods shows `<none>` in the Endpoints column
- The service selector must match the pod labels exactly
- Check `kubectl get pods -n fix-me --show-labels` to see actual pod labels
