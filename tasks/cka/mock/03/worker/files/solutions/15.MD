## Task 15: DNS Resolution — Save FQDNs

```bash
kubectl config use-context cluster1-admin@cluster1

# Get service FQDN and cluster IP
SVC_FQDN="dns-service.dns-test.svc.cluster.local"
SVC_IP=$(kubectl get svc dns-service -n dns-test -o jsonpath='{.spec.clusterIP}')

# Save service FQDN and IP to svc.txt
echo "${SVC_FQDN} ${SVC_IP}" > /var/work/tests/artifacts/15/svc.txt

# Get pod IP and construct pod DNS name
POD_IP=$(kubectl get pod dns-pod -n dns-test -o jsonpath='{.status.podIP}')
POD_IP_DASHED=$(echo "$POD_IP" | sed 's/\./-/g')
POD_FQDN="${POD_IP_DASHED}.dns-test.pod.cluster.local"

# Save pod FQDN and IP to pod.txt
echo "${POD_FQDN} ${POD_IP}" > /var/work/tests/artifacts/15/pod.txt

cat /var/work/tests/artifacts/15/svc.txt
cat /var/work/tests/artifacts/15/pod.txt
```

**Key concepts:**
- Service FQDN format: `<service-name>.<namespace>.svc.cluster.local`
- Pod DNS format: `<ip-dashed>.<namespace>.pod.cluster.local` (e.g., `10-0-1-5.dns-test.pod.cluster.local`)
- You can also resolve FQDNs by running nslookup from inside a pod:
  ```bash
  kubectl exec dns-pod -n dns-test -- nslookup dns-service
  ```
