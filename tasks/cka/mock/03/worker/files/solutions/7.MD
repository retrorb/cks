## Task 7: Deny-All Ingress NetworkPolicy

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all
  namespace: policy-ns
spec:
  podSelector: {}
  policyTypes:
  - Ingress
```

```bash
kubectl apply -f deny-all.yaml
```

**Verify:**
```bash
kubectl get networkpolicy deny-all -n policy-ns
# From external-ns, connection to target-app should time out
```

**Key concepts:**
- An empty `podSelector: {}` matches ALL pods in the namespace
- An empty (absent) `ingress: []` block with `policyTypes: [Ingress]` means NO ingress is allowed
- This creates a default-deny for all incoming traffic to the namespace
- Egress is not affected by this policy
