## Task 4: Headless Service

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: v1
kind: Service
metadata:
  name: db-headless
  namespace: data
spec:
  clusterIP: None
  selector:
    app: db
  ports:
  - port: 3306
    targetPort: 80
```

```bash
kubectl apply -f db-headless.yaml
```

**Key concepts:**
- Setting `clusterIP: None` makes a service "headless"
- Headless services do not allocate a cluster IP; DNS returns the pod IPs directly
- Essential for StatefulSets — each pod gets its own DNS entry: `<pod-name>.<service-name>.<namespace>.svc.cluster.local`
- The selector `app: db` matches the StatefulSet pods
