## Task 10: Egress NetworkPolicy

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: restrict-egress
  namespace: restricted-ns
spec:
  podSelector:
    matchLabels:
      app: restricted
  policyTypes:
  - Ingress
  - Egress
  egress:
  - ports:
    - port: 53
      protocol: UDP
    - port: 53
      protocol: TCP
  - to:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: allowed-target-ns
    ports:
    - port: 80
```

```bash
kubectl apply -f restrict-egress.yaml
```

**Key concepts:**
- `policyTypes: [Ingress, Egress]` with no `ingress:` block = deny all ingress
- Two egress rules: one for DNS (port 53), one for HTTP to allowed namespace
- Port 53 must be allowed for DNS to work (so service names resolve)
- Use `kubernetes.io/metadata.name` label (auto-set on all namespaces) for namespace selection
