## Task 16: Update CoreDNS with Custom Stub Zone

```bash
kubectl config use-context cluster1-admin@cluster1

# Edit the CoreDNS ConfigMap
kubectl edit configmap coredns -n kube-system
```

Add the stub zone block to the Corefile. The result should look like:

```
.:53 {
    errors
    health {
       lameduck 5s
    }
    ready
    kubernetes cluster.local in-addr.arpa ip6.arpa {
       pods insecure
       fallthrough in-addr.arpa ip6.arpa
       ttl 30
    }
    prometheus :9153
    forward . /etc/resolv.conf {
       max_concurrent 1000
    }
    cache 30
    loop
    reload
    loadbalance
}

custom.local:53 {
    forward . 8.8.8.8
    cache 30
}
```

```bash
# Restart CoreDNS pods to pick up the new config
kubectl rollout restart deployment coredns -n kube-system

# Verify CoreDNS pods come back healthy
kubectl rollout status deployment coredns -n kube-system
```

**Key concepts:**
- CoreDNS configuration is stored in a ConfigMap — edit it and restart the pods
- Stub zones forward specific domain queries to a custom DNS server
- The `custom.local` zone will forward all queries for `*.custom.local` to `8.8.8.8`
- CoreDNS uses the `reload` plugin to watch for ConfigMap changes, but restarting is safer
