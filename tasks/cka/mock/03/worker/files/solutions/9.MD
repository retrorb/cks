## Task 9: Cross-Namespace NetworkPolicy

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-trusted-namespace
  namespace: secure-ns
spec:
  podSelector:
    matchLabels:
      app: secure-server
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          team: trusted
```

```bash
kubectl apply -f allow-trusted-namespace.yaml
```

**Verify:**
```bash
# trusted-ns (label team=trusted) — should succeed:
kubectl exec trusted-client -n trusted-ns -- curl http://secure-server.secure-ns.svc -s --connect-timeout 3

# untrusted-ns (label team=untrusted) — should be blocked:
kubectl exec untrusted-client -n untrusted-ns -- curl http://secure-server.secure-ns.svc -s --connect-timeout 2
```

**Key concepts:**
- `namespaceSelector` selects source namespaces by their labels
- The namespace `trusted-ns` has label `team=trusted` — it passes
- The namespace `untrusted-ns` has label `team=untrusted` — it is blocked
- Label your own namespace: `kubectl label namespace trusted-ns team=trusted`
