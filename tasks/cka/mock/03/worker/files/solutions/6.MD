## Task 6: Pod + Service + Save Endpoint IP

```bash
kubectl config use-context cluster1-admin@cluster1

# Create pod
kubectl run endpoint-test --image=nginx:alpine --port=80

# Create service exposing the pod
kubectl expose pod endpoint-test \
  --name=endpoint-service \
  --port=80 \
  --target-port=80

# Wait for pod to be running
kubectl wait pod endpoint-test --for=condition=Ready --timeout=60s

# Get endpoint IP and save to file
kubectl get endpoints endpoint-service \
  -o jsonpath='{.subsets[0].addresses[0].ip}' \
  > /var/work/tests/artifacts/6/endpoints.txt

cat /var/work/tests/artifacts/6/endpoints.txt
```

**Key concepts:**
- Endpoints are automatically managed by the service controller
- The endpoint IP matches the pod's IP address
- `kubectl get endpoints <svc>` shows the current backing pod IPs
