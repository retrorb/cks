## Task 19: Multi-Port Service

```bash
kubectl config use-context cluster1-admin@cluster1
```

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: multi-port-pod
  namespace: default
  labels:
    app: multi-port-pod
spec:
  containers:
  - name: nginx
    image: nginx
    ports:
    - containerPort: 80
    - containerPort: 443
---
apiVersion: v1
kind: Service
metadata:
  name: multi-port-svc
  namespace: default
spec:
  selector:
    app: multi-port-pod
  ports:
  - name: http
    port: 80
    targetPort: 80
  - name: https
    port: 443
    targetPort: 443
```

```bash
kubectl apply -f multi-port.yaml

# Verify
kubectl get svc multi-port-svc -o yaml
```

**Key concepts:**
- When a service exposes multiple ports, each port **must** have a `name` field
- Port names must be unique within the service
- The `name` field is used by Ingress and other resources to reference specific ports
