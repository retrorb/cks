## Task 13: Fix Broken Ingress

```bash
kubectl config use-context cluster1-admin@cluster1

# Inspect the broken ingress
kubectl get ingress broken-ingress -n fix-ingress-ns -o yaml
# The backend service is "wrong-svc" — should be "fix-app-svc"

# Fix the service name
kubectl patch ingress broken-ingress -n fix-ingress-ns \
  --type='json' \
  -p='[{"op":"replace","path":"/spec/rules/0/http/paths/0/backend/service/name","value":"fix-app-svc"}]'
```

Or edit directly:
```bash
kubectl edit ingress broken-ingress -n fix-ingress-ns
# Change backend.service.name from "wrong-svc" to "fix-app-svc"
```

**Verify:**
```bash
kubectl get ingress broken-ingress -n fix-ingress-ns -o jsonpath='{.spec.rules[0].http.paths[0].backend.service.name}'
# Should output: fix-app-svc
```

**Key concepts:**
- Ingress routing breaks silently when the backend service name is wrong
- Check `kubectl describe ingress <name>` for backend service validation warnings
- The ingress controller logs will show 503 errors when the service doesn't exist
