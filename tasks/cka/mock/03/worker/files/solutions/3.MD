## Task 3: LoadBalancer Service

```bash
kubectl config use-context cluster1-admin@cluster1

kubectl expose pod api-pod \
  --name=api-lb \
  --port=80 \
  --target-port=8080 \
  --type=LoadBalancer
```

Or using a manifest:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: api-lb
  namespace: default
spec:
  type: LoadBalancer
  selector:
    app: api-pod
  ports:
  - port: 80
    targetPort: 8080
```

**Key concepts:**
- LoadBalancer provisions an external load balancer (cloud provider)
- On bare metal/kubeadm clusters, LoadBalancer services stay in `<pending>` state for the external IP — this is expected
- The `port` is what clients connect to; `targetPort` is the container port (8080 for ping_pong)
