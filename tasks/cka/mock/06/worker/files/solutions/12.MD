# Task 12 — Fix stuck-pod by removing nodeSelector in affinity-ns (4 pts)

## Problem
Pod `stuck-pod` in namespace `affinity-ns` has `nodeSelector: disk: ssd` which matches no nodes, leaving it in Pending state.

## Solution

Since nodeSelector cannot be updated on a running pod, delete and recreate:
```bash
kubectl get pod stuck-pod -n affinity-ns --context cluster1-admin@cluster1 -o yaml > /tmp/stuck-pod.yaml
```

Edit `/tmp/stuck-pod.yaml` and remove the `nodeSelector` section entirely.

Apply:
```bash
kubectl delete pod stuck-pod -n affinity-ns --context cluster1-admin@cluster1
kubectl apply -f /tmp/stuck-pod.yaml --context cluster1-admin@cluster1
```

Verify:
```bash
kubectl get pod stuck-pod -n affinity-ns --context cluster1-admin@cluster1
```
