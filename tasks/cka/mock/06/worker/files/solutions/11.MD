# Task 11 — Fix ConfigMap app-config DB_HOST in config-ns (5 pts)

## Problem
ConfigMap `app-config` in namespace `config-ns` has `DB_HOST: wrong-host`. Pod `env-pod` reads this value as an environment variable.

## Solution

```bash
kubectl patch configmap app-config -n config-ns --context cluster1-admin@cluster1 \
  --type merge -p '{"data":{"DB_HOST":"correct-db-host"}}'
```

Restart the pod to pick up the updated ConfigMap value:
```bash
kubectl delete pod env-pod -n config-ns --context cluster1-admin@cluster1
```

The pod will be recreated automatically if managed by a controller, otherwise recreate manually:
```bash
kubectl run env-pod -n config-ns --image=nginx:alpine --context cluster1-admin@cluster1 \
  --env="DB_HOST=$(kubectl get cm app-config -n config-ns --context cluster1-admin@cluster1 -o jsonpath='{.data.DB_HOST}')"
```

Or edit and recreate with valueFrom:
```bash
kubectl get pod env-pod -n config-ns --context cluster1-admin@cluster1 -o yaml > /tmp/env-pod.yaml
kubectl delete pod env-pod -n config-ns --context cluster1-admin@cluster1
kubectl apply -f /tmp/env-pod.yaml --context cluster1-admin@cluster1
```

Verify:
```bash
kubectl exec env-pod -n config-ns --context cluster1-admin@cluster1 -- env | grep DB_HOST
```
