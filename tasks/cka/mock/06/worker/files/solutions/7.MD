# Task 7 — Fix NetworkPolicy block-egress to allow DNS (5 pts)

## Problem
NetworkPolicy `block-egress` in namespace `dns-block` blocks all egress traffic including DNS (port 53), preventing pods from resolving names.

## Solution

```bash
kubectl edit networkpolicy block-egress -n dns-block --context cluster1-admin@cluster1
```

Update the egress section to allow DNS:
```yaml
spec:
  podSelector: {}
  policyTypes:
  - Egress
  egress:
  - ports:
    - port: 53
      protocol: UDP
    - port: 53
      protocol: TCP
```

Or apply with kubectl:
```bash
kubectl apply -f - --context cluster1-admin@cluster1 <<EOF
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: block-egress
  namespace: dns-block
spec:
  podSelector: {}
  policyTypes:
  - Egress
  egress:
  - ports:
    - port: 53
      protocol: UDP
    - port: 53
      protocol: TCP
EOF
```

Verify:
```bash
kubectl exec no-dns-pod -n dns-block --context cluster1-admin@cluster1 \
  -- nslookup kubernetes.default.svc.cluster.local
```
