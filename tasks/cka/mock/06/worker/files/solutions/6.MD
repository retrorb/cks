# Task 6 — Fix no-ep-svc selector in ep-ns (5 pts)

## Problem
Service `no-ep-svc` in namespace `ep-ns` has selector `app=wrong-app` but the actual pods have label `app=real-app`, so the service has no endpoints.

## Solution

```bash
kubectl patch svc no-ep-svc -n ep-ns --context cluster1-admin@cluster1 \
  --type='json' -p='[{"op":"replace","path":"/spec/selector/app","value":"real-app"}]'
```

Or edit directly:
```bash
kubectl edit svc no-ep-svc -n ep-ns --context cluster1-admin@cluster1
# Change: app: wrong-app  →  app: real-app
```

Verify:
```bash
kubectl get endpoints no-ep-svc -n ep-ns --context cluster1-admin@cluster1
```
