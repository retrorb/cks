# Task 2 — Fix extra-scheduler static pod on cluster2 (5 pts)

## Problem
The static pod `extra-scheduler` in `/etc/kubernetes/manifests/extra-scheduler.yaml` on the cluster2 master uses `--config=/nonexistent/scheduler-config.yaml`. The correct config file is at `/etc/kubernetes/scheduler-config.yaml`.

## Solution

SSH to the cluster2 master node:
```bash
MASTER2=$(kubectl get nodes --context cluster2-admin@cluster2 \
  -l 'node-role.kubernetes.io/control-plane' \
  -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')
ssh ubuntu@$MASTER2
```

On the master node:
```bash
sudo sed -i 's|/nonexistent/scheduler-config.yaml|/etc/kubernetes/scheduler-config.yaml|g' \
  /etc/kubernetes/manifests/extra-scheduler.yaml
```

The kubelet will automatically restart the static pod. Verify:
```bash
kubectl get pods -n kube-system --context cluster2-admin@cluster2 | grep extra-scheduler
```
