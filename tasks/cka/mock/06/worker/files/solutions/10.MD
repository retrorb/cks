# Task 10 — Fix oom-pod memory limit in resource-ns (4 pts)

## Problem
Pod `oom-pod` in namespace `resource-ns` has a memory limit of `1Mi` which is too low for nginx, causing OOMKill restarts.

## Solution

Since Pod resource limits cannot be updated in-place, delete and recreate:
```bash
kubectl get pod oom-pod -n resource-ns --context cluster1-admin@cluster1 -o yaml > /tmp/oom-pod.yaml
```

Edit `/tmp/oom-pod.yaml` and change memory limit from `1Mi` to `64Mi`:
```yaml
resources:
  limits:
    memory: "64Mi"
  requests:
    memory: "64Mi"
```

Apply:
```bash
kubectl delete pod oom-pod -n resource-ns --context cluster1-admin@cluster1
kubectl apply -f /tmp/oom-pod.yaml --context cluster1-admin@cluster1
```

Verify:
```bash
kubectl get pod oom-pod -n resource-ns --context cluster1-admin@cluster1
```
