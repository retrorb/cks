# Task 1 — Fix cluster2 worker node NotReady (5 pts)

## Problem
The kubelet on the cluster2 worker node references `/var/lib/kubelet/config-broken.yaml` instead of `/var/lib/kubelet/config.yaml`.

## Solution

SSH to the cluster2 worker node:
```bash
# Get the cluster2 worker IP
WORKER2=$(kubectl get nodes --context cluster2-admin@cluster2 -l work_type=infra_core \
  -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')
ssh ubuntu@$WORKER2
```

On the worker node:
```bash
sudo sed -i 's|/var/lib/kubelet/config-broken.yaml|/var/lib/kubelet/config.yaml|g' \
  /etc/systemd/system/kubelet.service.d/10-kubeadm.conf

sudo systemctl daemon-reload
sudo systemctl restart kubelet
```

Verify:
```bash
sudo systemctl status kubelet
kubectl get nodes --context cluster2-admin@cluster2
```
