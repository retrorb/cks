# Task 19 — Add nodeAffinity + CPU/memory limits + livenessProbe to affinity-app (8 pts)

## Solution

Edit the deployment:
```bash
kubectl edit deployment affinity-app -n schedule-ns --context cluster1-admin@cluster1
```

Add the following to the deployment spec:

```yaml
spec:
  template:
    spec:
      affinity:
        nodeAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 1
            preference:
              matchExpressions:
              - key: zone
                operator: In
                values:
                - primary
      containers:
      - name: affinity-app
        image: nginx:alpine
        resources:
          limits:
            cpu: "200m"
            memory: "256Mi"
        livenessProbe:
          httpGet:
            path: /healthz
            port: 8080
          periodSeconds: 10
```

Or patch with kubectl:
```bash
kubectl patch deployment affinity-app -n schedule-ns --context cluster1-admin@cluster1 \
  --type=json -p='[
    {"op":"add","path":"/spec/template/spec/affinity","value":{"nodeAffinity":{"preferredDuringSchedulingIgnoredDuringExecution":[{"weight":1,"preference":{"matchExpressions":[{"key":"zone","operator":"In","values":["primary"]}]}}]}}},
    {"op":"add","path":"/spec/template/spec/containers/0/resources","value":{"limits":{"cpu":"200m","memory":"256Mi"}}},
    {"op":"add","path":"/spec/template/spec/containers/0/livenessProbe","value":{"httpGet":{"path":"/healthz","port":8080},"periodSeconds":10}}
  ]'
```

Verify:
```bash
kubectl get deployment affinity-app -n schedule-ns --context cluster1-admin@cluster1 -o yaml | grep -A 20 affinity
kubectl get deployment affinity-app -n schedule-ns --context cluster1-admin@cluster1 -o yaml | grep -A 5 livenessProbe
```
