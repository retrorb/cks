# Task 4 — Fix crash-app pod in crash-ns (5 pts)

## Problem
Pod `crash-app` in namespace `crash-ns` is in CrashLoopBackOff because its command is set to `/bin/wrong` which does not exist.

## Solution

Delete and recreate the pod with a valid command:
```bash
kubectl delete pod crash-app -n crash-ns --context cluster1-admin@cluster1

kubectl run crash-app -n crash-ns --image=nginx:alpine \
  --context cluster1-admin@cluster1
```

Or edit the existing pod (which requires delete + recreate for command changes):
```bash
kubectl get pod crash-app -n crash-ns --context cluster1-admin@cluster1 -o yaml > /tmp/crash-app.yaml
# Edit /tmp/crash-app.yaml: remove the command field
kubectl delete pod crash-app -n crash-ns --context cluster1-admin@cluster1
kubectl apply -f /tmp/crash-app.yaml --context cluster1-admin@cluster1
```

Verify:
```bash
kubectl get pod crash-app -n crash-ns --context cluster1-admin@cluster1
```
