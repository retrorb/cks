## Task 1: Create Role, SA, and RoleBinding

```bash
kubectl config use-context cluster1-admin@cluster1

# Create Role
kubectl create role pod-reader -n rbac-ns \
  --verb=get,list \
  --resource=pods

# Create RoleBinding
kubectl create rolebinding pod-reader-binding -n rbac-ns \
  --role=pod-reader \
  --serviceaccount=rbac-ns:reader-sa
```

**Key concepts:**
- Roles are namespace-scoped; ClusterRoles are cluster-scoped
- RoleBindings bind a Role to subjects (users, groups, service accounts)
- Use `kubectl auth can-i` to verify permissions
