## Task 4: Create SA, CRB, and Save can-i Result

```bash
kubectl config use-context cluster1-admin@cluster1

# Create SA
kubectl create serviceaccount check-sa -n check-ns

# Create ClusterRoleBinding to view ClusterRole
kubectl create clusterrolebinding check-crb \
  --clusterrole=view \
  --serviceaccount=check-ns:check-sa

# Save can-i result
kubectl auth can-i list pods \
  --as=system:serviceaccount:check-ns:check-sa \
  > /var/work/tests/artifacts/4/can-i.txt
```

**Key concepts:**
- The `view` ClusterRole is built-in and allows read-only access to most resources
- `--as=system:serviceaccount:namespace:name` impersonates a SA for can-i checks
