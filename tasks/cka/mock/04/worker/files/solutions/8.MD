## Task 8: Backup etcd Snapshot

```bash
kubectl config use-context cluster1-admin@cluster1

# SSH to master and run etcdctl snapshot save
MASTER=$(kubectl get nodes -o jsonpath='{.items[?(@.metadata.labels.node-role\.kubernetes\.io/control-plane)].status.addresses[0].address}')

mkdir -p /var/work/tests/artifacts/8

ssh -o StrictHostKeyChecking=no ubuntu@$MASTER \
  "sudo ETCDCTL_API=3 etcdctl \
    --endpoints=https://127.0.0.1:2379 \
    --cacert=/etc/kubernetes/pki/etcd/ca.crt \
    --cert=/etc/kubernetes/pki/etcd/server.crt \
    --key=/etc/kubernetes/pki/etcd/server.key \
    snapshot save /var/work/tests/artifacts/8/etcd-backup.db"

# Verify
ls -lh /var/work/tests/artifacts/8/etcd-backup.db
```

**Key concepts:**
- `ETCDCTL_API=3` is required for etcd v3
- Always specify `--endpoints`, `--cacert`, `--cert`, `--key`
- Verify with `etcdctl snapshot status <file>`
