## Task 20: Recreate CR in Correct Namespace

```bash
kubectl config use-context cluster1-admin@cluster1

# View the existing CR in default namespace
kubectl get backup my-backup -n default -o yaml

# Delete the CR from default namespace
kubectl delete backup my-backup -n default

# Recreate in backup-ns
cat << 'EOF' | kubectl apply -f -
apiVersion: storage.example.com/v1
kind: Backup
metadata:
  name: my-backup
  namespace: backup-ns
spec:
  target: etcd
EOF

# Verify
kubectl get backup my-backup -n backup-ns
```

**Key concepts:**
- CRs cannot be moved between namespaces; must delete and recreate
- The apiVersion is `group/version` from the CRD definition
- Always verify the namespace is correct after recreation
