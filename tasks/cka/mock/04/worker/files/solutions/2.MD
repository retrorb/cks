## Task 2: Create ClusterRole and ClusterRoleBinding

```bash
kubectl config use-context cluster1-admin@cluster1

# Create ClusterRole
kubectl create clusterrole node-reader \
  --verb=get,list,watch \
  --resource=nodes

# Create ClusterRoleBinding
kubectl create clusterrolebinding node-reader-binding \
  --clusterrole=node-reader \
  --serviceaccount=monitoring:node-monitor-sa
```

**Key concepts:**
- ClusterRoles apply cluster-wide; they can see cluster-scoped resources like nodes
- ClusterRoleBindings can reference SAs from any namespace using `namespace:name` format
