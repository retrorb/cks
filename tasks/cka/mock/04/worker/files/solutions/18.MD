## Task 18: Find etcd-servers Flag

```bash
kubectl config use-context cluster1-admin@cluster1

# SSH to master and read the static pod manifest
MASTER=$(kubectl get nodes -o jsonpath='{.items[?(@.metadata.labels.node-role\.kubernetes\.io/control-plane)].status.addresses[0].address}')

mkdir -p /var/work/tests/artifacts/18

ssh -o StrictHostKeyChecking=no ubuntu@$MASTER \
  "sudo grep -i etcd-servers /etc/kubernetes/manifests/kube-apiserver.yaml" \
  | grep -oP '(?<=--etcd-servers=)\S+' \
  > /var/work/tests/artifacts/18/etcd-url.txt

# Alternative: show value directly
cat /var/work/tests/artifacts/18/etcd-url.txt
```

**Key concepts:**
- Static pods are defined in `/etc/kubernetes/manifests/`
- kube-apiserver connects to etcd using the `--etcd-servers` flag
- Default etcd endpoint is `https://127.0.0.1:2379`
