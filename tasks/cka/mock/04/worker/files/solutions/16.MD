## Task 16: Identify CNI Plugin

```bash
kubectl config use-context cluster1-admin@cluster1

# Method 1: Check CNI config files on node
# SSH to a node and look at /etc/cni/net.d/

# Method 2: Check running pods in kube-system
kubectl get pods -n kube-system --context cluster1-admin@cluster1 | grep -i calico

# Method 3: Check node annotations
kubectl describe node $(kubectl get nodes -o jsonpath='{.items[0].name}') | grep cni

# Save result
echo "calico" > /var/work/tests/artifacts/16/cni.txt
```

**Key concepts:**
- CNI (Container Network Interface) plugins manage pod networking
- Common CNIs: Calico, Cilium, Flannel, Weave
- Can be identified by pods in kube-system or config files in /etc/cni/net.d/
