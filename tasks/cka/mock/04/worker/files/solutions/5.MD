## Task 5: Create Role with CRUD on Deployments

```bash
kubectl config use-context cluster1-admin@cluster1

# Create Role
kubectl create role deploy-manager -n deploy-rbac \
  --verb=create,get,list,update,patch,delete \
  --resource=deployments

# Create RoleBinding
kubectl create rolebinding deploy-rb -n deploy-rbac \
  --role=deploy-manager \
  --serviceaccount=deploy-rbac:deploy-sa
```

**Key concepts:**
- Full CRUD requires: create, get, list, update, patch, delete
- `update` modifies existing resources; `patch` applies partial updates
